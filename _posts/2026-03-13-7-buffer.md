# 7 Buffer 方案详解

## 1. 概述

### 1.1 什么是 7 Buffer

7 Buffer 是昇腾 NPU 上针对 **MLA (Multi-head Latent Attention)** 场景设计的 L1 buffer 分配方案。

在计算 Attention 的矩阵乘 `Q × K^T` 时：
- **4 个 buffer** 存储 Q 矩阵的切块（称为 QP Buffer）
- **3 个 buffer** 存储 K 矩阵的切块（称为 KV Buffer）

每个 buffer 大小为 **72KB**，共计 **504KB**，几乎占满昇腾 910B 的 512KB L1。

### 1.2 适用场景

| 参数 | 值 | 说明 |
|------|-----|------|
| headDim | 512 | MLA nope 维度 |
| headDimRope | 64 | MLA rope 维度 |
| K 总维度 | 576 | 512 + 64 |
| 数据类型 | bf16 | 2 Bytes |

### 1.3 核心思想

```
┌─────────────────────────────────────────────────────────────┐
│                     7 Buffer 设计理念                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Q 矩阵 (4 块): 加载一次，常驻 L1，被多次复用               │
│  K 矩阵 (3 块): 流式处理，Triple Buffer 轮换                │
│                                                             │
│  目标: 让数据搬运与 Cube 计算完全重叠，消除等待             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 2. 背景知识

### 2.1 昇腾存储层次

理解 7 Buffer 需要先了解昇腾的存储层次：

```
┌─────────────────────────────────────────────────────────────┐
│                    昇腾存储层次                              │
├─────────────┬──────────┬───────────┬────────────────────────┤
│   存储层级  │   容量   │   带宽    │        用途            │
├─────────────┼──────────┼───────────┼────────────────────────┤
│ GM (HBM)    │  64 GB   │ 1.5 TB/s  │ 全局存储，容量大延迟高 │
│ L1          │ 512 KB   │ ~10 TB/s  │ 片上缓存，7 Buffer 所在│
│ L0A/L0B     │  64 KB   │ ~20 TB/s  │ Cube 输入缓存          │
│ L0C         │ 128 KB   │ ~20 TB/s  │ Cube 输出缓存          │
└─────────────┴──────────┴───────────┴────────────────────────┘

数据流: GM → L1 → L0A/L0B → Cube → L0C → GM
```

### 2.2 为什么需要切分

原始矩阵太大，无法一次放入 L1：

```
Q 矩阵: 256 × 576 × 2B = 288 KB
K 矩阵: 512 × 576 × 2B = 576 KB
总计: 864 KB > 512 KB (L1 容量)

解决方案: 将矩阵切成小块，分批加载计算
```

## 3. 为什么是 7 Buffer

### 3.1 硬件约束

```
L1 容量: 512 KB
单块大小: 128 行 × 288 列 × 2B = 72 KB

最多容纳: 512 / 72 ≈ 7 块
实际使用: 4 (QP) + 3 (KV) = 7 块
占用: 7 × 72 = 504 KB
剩余: 8 KB
```

### 3.2 Q 为什么是 4 块

Q 矩阵被切成 **2 × 2 = 4 块**（M 方向 2 块 × K 方向 2 块）：

```
Q 矩阵 [256, 576] 的切分:

           kL1=0 (288列)        kL1=1 (288列)
         ┌─────────────────┬─────────────────┐
 mL1=0   │     Q[0]        │     Q[1]        │  128行
[0:127]  │    ka=0         │    ka=1         │
         ├─────────────────┼─────────────────┤
 mL1=1   │     Q[2]        │     Q[3]        │  128行
[128:255]│    ka=2         │    ka=3         │
         └─────────────────┴─────────────────┘

关键点:
- 4 块正好对应 4 个 QP Buffer
- Q 只需加载一次，此后常驻 L1
- 被 4 轮 nL1 循环复用（复用率 4x）
```

### 3.3 K 为什么是 3 块

K 矩阵被切成 **4 × 2 = 8 块**，但只用 **3 个 buffer 轮换**：

```
K 矩阵 [512, 576] 的切分:

           kL1=0 (288列)        kL1=1 (288列)
         ┌─────────────────┬─────────────────┐
 nL1=0   │    K[0,0]       │    K[0,1]       │  128行
         ├─────────────────┼─────────────────┤
 nL1=1   │    K[1,0]       │    K[1,1]       │  128行
         ├─────────────────┼─────────────────┤
 nL1=2   │    K[2,0]       │    K[2,1]       │  128行
         ├─────────────────┼─────────────────┤
 nL1=3   │    K[3,0]       │    K[3,1]       │  128行
         └─────────────────┴─────────────────┘

为什么 3 块足够？
- K 是流式处理：搬运 → 计算 → 丢弃
- 3 块实现 Triple Buffer：搬运/计算/空闲 三态轮转
- 搬运与计算完全并行，无需等待
```

### 3.4 Q 和 K 块数差异的本质

| 对比项 | Q 矩阵 (4块) | K 矩阵 (3块) |
|--------|-------------|-------------|
| 访问模式 | 常驻复用 | 流式处理 |
| 数据生命周期 | 全程保留 | 用完即弃 |
| 块数决定因素 | 数据依赖 (mL1×kL1=2×2) | 流水线深度 (3态轮转) |

## 5. 计算流程

### 5.1 循环嵌套结构

```
┌─────────────────────────────────────────────────────────────────────────┐
│ ComputeMm1: Q[256,576] × K[512,576]^T → Score[256,512]                  │
│                                                                         │
│ ┌─────────────────────────────────────────────────────────────────────┐ │
│ │ for nL1 in [0,4):  // 512 tokens ÷ 128，遍历 K 的行方向              │ │
│ │                                                                     │ │
│ │ ┌─────────────────────────────────────────────────────────────────┐ │ │
│ │ │ for kL1 in [0,2):  // 576 dims ÷ 288，遍历 K 维度                │ │ │
│ │ │                                                                 │ │ │
│ │ │   【搬运 K】GM → L1，使用 Triple Buffer 轮换                     │ │ │
│ │ │                                                                 │ │ │
│ │ │ ┌─────────────────────────────────────────────────────────────┐ │ │ │
│ │ │ │ for mL1 in [0,2):  // 256 行 ÷ 128，遍历 Q 的行方向          │ │ │ │
│ │ │ │                                                             │ │ │ │
│ │ │ │   【搬运 Q】GM → L1（仅 nL1==0 时执行）                       │ │ │ │
│ │ │ │                                                             │ │ │ │
│ │ │ │ ┌─────────────────────────────────────────────────────────┐ │ │ │ │
│ │ │ │ │ for kL0 in [0,3):  // 288 dims ÷ 96，L0 级切分           │ │ │ │ │
│ │ │ │ │                                                         │ │ │ │ │
│ │ │ │ │   Load3D: L1 → L0A/L0B                                  │ │ │ │ │
│ │ │ │ │   Mmad: Q[128,96] × K[128,96]^T，累加到 L0C              │ │ │ │ │
│ │ │ │ │                                                         │ │ │ │ │
│ │ │ │ └─────────────────────────────────────────────────────────┘ │ │ │ │
│ │ │ │                                                             │ │ │ │
│ │ │ │   【输出】L0C → GM（仅 kL1==1 时，K维度累加完成后输出）       │ │ │ │
│ │ │ │                                                             │ │ │ │
│ │ │ └─────────────────────────────────────────────────────────────┘ │ │ │
│ │ └─────────────────────────────────────────────────────────────────┘ │ │
│ └─────────────────────────────────────────────────────────────────────┘ │
│                                                                         │
│ 总计: 4 × 2 × 2 × 3 = 48 次 Mmad                                        │
└─────────────────────────────────────────────────────────────────────────┘
```

### 5.2 循环次数汇总

| 循环层 | 变量 | 次数 | 切分维度 | 说明 |
|--------|------|------|----------|------|
| 第1层 | nL1 | 4 | K 的行方向 | 512 ÷ 128 |
| 第2层 | kL1 | 2 | K 维度 | 576 ÷ 288 |
| 第3层 | mL1 | 2 | Q 的行方向 | 256 ÷ 128 |
| 第4层 | kL0 | 3 | L0 级 K 维度 | 288 ÷ 96 |

### 5.3 KV Buffer 轮换详解

```
nL1 × kL1 循环展开 (共 8 次迭代):

┌───────┬──────┬─────────────┬──────────────────────────────────────────────┐
│ nL1   │ kL1  │ kvL1BufIter │ KV Buffer 状态                               │
├───────┼──────┼─────────────┼──────────────────────────────────────────────┤
│   0   │  0   │     0       │ KV[0]=搬K[0,0]  KV[1]=空       KV[2]=空      │
│   0   │  1   │     1       │ KV[0]=计算中    KV[1]=搬K[0,1]  KV[2]=空     │
├───────┼──────┼─────────────┼──────────────────────────────────────────────┤
│   1   │  0   │     2       │ KV[0]=空        KV[1]=计算中    KV[2]=搬K[1,0]│
│   1   │  1   │     3       │ KV[0]=搬K[1,1]  KV[1]=空        KV[2]=计算中 │
├───────┼──────┼─────────────┼──────────────────────────────────────────────┤
│   2   │  0   │     4       │ KV[0]=计算中    KV[1]=搬K[2,0]  KV[2]=空     │
│   2   │  1   │     5       │ KV[0]=空        KV[1]=计算中    KV[2]=搬K[2,1]│
├───────┼──────┼─────────────┼──────────────────────────────────────────────┤
│   3   │  0   │     6       │ KV[0]=搬K[3,0]  KV[1]=空        KV[2]=计算中 │
│   3   │  1   │     7       │ KV[0]=计算中    KV[1]=搬K[3,1]  KV[2]=空     │
└───────┴──────┴─────────────┴──────────────────────────────────────────────┘

选择公式: kb = kvL1BufIter % 3
轮转模式: 搬运 → 计算 → 空闲 → 搬运 ...
```

## 6. 循环顺序分析

### 6.1 为什么是 n-k-m 顺序

当前循环顺序：**nL1 → kL1 → mL1**（从外到内）

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        循环顺序设计原理                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   nL1 (最外层): 遍历 K 的行方向                                          │
│        ↓                                                                │
│        │  Q 仅在 nL1=0 时加载，之后常驻 L1                               │
│        │  每轮 nL1 复用同一份 Q → 4x 复用率                              │
│        ↓                                                                │
│   kL1 (中间层): 遍历 K 维度                                              │
│        ↓                                                                │
│        │  K 流式加载，用 Triple Buffer 隐藏搬运延迟                      │
│        │  同一 K 块被所有 mL1 迭代复用后再丢弃                           │
│        ↓                                                                │
│   mL1 (最内层): 遍历 Q 的行方向                                          │
│        ↓                                                                │
│        │  每个 K 块与所有 Q 块完成计算后再换下一个 K                     │
│        │  最大化每块 K 的利用率                                         │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

**核心原则**：

| 循环位置 | 放什么 | 原因 |
|----------|--------|------|
| 最外层 | N (K行) | Q 加载一次，跨所有 nL1 复用 |
| 中间层 | K维度 | 与 L0C 累加对齐，一次完整 K 维累加后输出 |
| 最内层 | M (Q行) | 每块 K 计算完所有 Q 块后再换 |

### 6.2 替代方案分析

6 种可能的循环顺序对比：

```
┌────────────┬─────────────────────────────────────────────────────────────┐
│   顺序     │                       分析                                  │
├────────────┼─────────────────────────────────────────────────────────────┤
│ n-k-m ✓   │ 当前方案。Q 常驻复用 4x，K 流式 Triple Buffer               │
│            │ Q 搬 4 次，K 搬 8 次                                        │
├────────────┼─────────────────────────────────────────────────────────────┤
│ n-m-k      │ K 维度最内层，每次 mL1 迭代内完成 K 累加                    │
│            │ 问题：K 的两块 (kL1=0,1) 需同时在 L1，buffer 需求翻倍       │
├────────────┼─────────────────────────────────────────────────────────────┤
│ k-n-m      │ K 维度最外层                                                │
│            │ 问题：需在 L0C 保留部分和，跨 kL1 累加，L0C 容量不足        │
├────────────┼─────────────────────────────────────────────────────────────┤
│ k-m-n      │ 类似上条，且 Q 复用率降低                                   │
│            │ 问题：同上，L0C 不足                                        │
├────────────┼─────────────────────────────────────────────────────────────┤
│ m-n-k      │ Q 行最外层                                                  │
│            │ 问题：每轮 mL1 需重新加载 K，K 搬 16 次 (2x 开销)           │
├────────────┼─────────────────────────────────────────────────────────────┤
│ m-k-n      │ Q 行最外层，K 维中间                                        │
│            │ 问题：每轮 nL1 需重载 Q，Q 搬 32 次 (8x 开销)               │
└────────────┴─────────────────────────────────────────────────────────────┘
```

### 6.3 关键约束

决定循环顺序的三个硬件/算法约束：

```
约束1: L0C 累加 (K 维度必须连续完成)
─────────────────────────────────────
  Q × K^T 的 K 维度 = 576，切成 2 块 (kL1=0,1)
  L0C 只有 128KB，只能存一个 [128,128] 的部分和

  → kL1=0 和 kL1=1 必须连续计算，完成累加后才能输出
  → K 维度不能放最外层

约束2: L1 容量 (Q 和 K 的 buffer 数量有限)
─────────────────────────────────────
  L1 = 512KB，Q 用 4 块，K 用 3 块

  → Q 必须常驻：加载一次，被多轮循环复用
  → K 必须流式：用完就覆盖，不能全部常驻
  → N (K行) 方向放最外层，让 Q 跨 nL1 复用

约束3: Triple Buffer 流水 (搬运与计算并行)
─────────────────────────────────────
  3 块 KV Buffer 实现：搬运 → 计算 → 空闲 → 搬运...

  → K 搬运在 kL1 循环层
  → 内层 mL1 循环完成计算，保证 buffer 占用时间充分
  → mL1 放最内层配合 Triple Buffer
```

### 6.4 总结

**n-k-m 是唯一满足所有约束的顺序**：

| 约束 | 要求 | n-k-m 如何满足 |
|------|------|---------------|
| L0C 累加 | K 维连续 | kL1 是连续的中间循环 |
| Q 复用 | 常驻 L1 | nL1 最外层，Q 只加载一次 |
| K 流式 | Triple Buffer | kL1 每次迭代搬运新 K |
| 带宽平衡 | 最小化搬运 | Q 搬 4 次，K 搬 8 次，总 12 次 |

任何其他顺序都会违反至少一条约束，导致性能下降或无法实现。

## 7. 关键优化技巧

### 7.1 Q 矩阵常驻复用

Q 只在第一轮 nL1 循环时加载，之后常驻 L1：

```cpp
// 代码体现
if (nL1 == 0) {  // 仅首轮加载
    CopyInMm1AToL1(aL1Tensor, ...);
}
// nL1=1,2,3 直接复用 L1 中的 Q
```

**收益分析**：
- 搬运次数：4 块 × 1 次 = 4 次
- 使用次数：4 块 × 4 轮 = 16 次
- 复用率：**4x**

### 7.2 K 矩阵 Triple Buffer

通过事件标志实现同步：

```cpp
// 每次 kL1 迭代
kvL1BufIter++;
kb = kvL1BufIter % 3;                      // 选择 buffer

WaitFlag<MTE1_MTE2>(mte21KVIds[kb]);       // 1. 等待 buffer 空闲
DataCopy(bL1Tensor, kvMergeGm_[...]);      // 2. 搬运数据
SetFlag<MTE2_MTE1>(mte21KVIds[kb]);        // 3. 通知搬运完成

for (mL1...) { Mmad(...); }                // 4. 计算

SetFlag<MTE1_MTE2>(mte21KVIds[kb]);        // 5. 通知 buffer 可覆盖
```

**流水线效果**：
```
时间 →   T0      T1      T2      T3      T4      T5
KV[0]   搬运    计算    空闲    搬运    计算    空闲
KV[1]   空闲    搬运    计算    空闲    搬运    计算
KV[2]   空闲    空闲    搬运    计算    空闲    搬运
        ─────────────────────────────────────────────
        每时刻：1块搬运 + 1块计算，流水线满载
```

### 7.3 QP Buffer 连续性映射

通过索引映射确保同一 Q 行的 K 维度数据物理连续：

```cpp
uint32_t GetQPL1RealIdx(uint32_t mIdx, uint32_t kL1Idx) {
    uint32_t idxMap[] = {0, 2};  // 关键设计
    return idxMap[mIdx % 2] + kL1Idx;
}

// 映射结果:
// mIdx=0: QP[0], QP[1] 相邻 → Q[0:127] 的 576 列连续
// mIdx=1: QP[2], QP[3] 相邻 → Q[128:255] 的 576 列连续
```

**好处**：Load3D 访问完整 K 维度时，内存地址连续，带宽利用率高。

### 7.4 rope 跨块连续设计

kL1=0 和 kL1=1 的数据排布顺序不同，确保 rope 部分物理连续：

```
kL1=0 布局:                    kL1=1 布局:
┌──────────────┬────────┐      ┌────────┬──────────────┐
│ nope [0:255] │rope[0:31]│    │rope[32:63]│ nope [256:511]│
│   256 列     │  32 列  │      │  32 列  │    256 列    │
└──────────────┴────────┘      └────────┴──────────────┘
    nope 在前                      rope 在前

结果: rope 的完整 64 列物理连续
```

### 7.5 优化收益汇总

| 优化技巧 | 作用 | 收益 |
|----------|------|------|
| Q 常驻复用 | 减少 GM 访问 | 4x 数据复用 |
| K Triple Buffer | 隐藏搬运延迟 | ~1.4x 加速 |
| QP 连续性映射 | 提高带宽利用 | Load3D 效率提升 |
| rope 跨块连续 | 避免数据拆分 | 简化计算逻辑 |

## 8. 常见问题

### Q1: 如果 L1 更大，K 应该用 4 块吗？

**不需要**。3 块已经是最优解。

Triple Buffer 的流水线效率与 Quad Buffer 相同（每时刻 1 搬运 + 1 计算），多出的 buffer 只会空闲，浪费内存。

如果 L1 有富余，更好的优化方向：
- 增大 `M_SPLIT_SIZE` 或 `N_SPLIT_SIZE`，减少循环开销
- 增大 `kL1Size`，减少 K 维度切分次数

### Q2: 这个方案能用于非 MLA 场景吗？

需要调整。7 Buffer 的参数（72KB/块、288 列/块）是针对 MLA 的 576 维 K 优化的。不同的 headDim 需要重新计算切分策略。

---

## 9. 总结

7 Buffer 方案的核心设计：

1. **Q (4块)**: 常驻复用，减少 GM 访问
2. **K (3块)**: Triple Buffer 流式处理，隐藏搬运延迟
3. **504KB / 512KB**: 极限利用 L1 容量
4. **搬运与计算完全重叠**: 流水线满载运行

这是一个针对 MLA 场景高度优化的内存分配方案，通过精细的切分策略和流水线设计，最大化昇腾 NPU 的计算效率。
